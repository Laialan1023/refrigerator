# refrigerator
冰箱多物品冷卻模擬

一 程式功能與技術原理

本程式用來模擬多個物品放入冰箱後的冷卻行為。使用者可輸入冰箱溫度與各物品的物理參數，並依需求選擇兩種功能模式。第一種模式為已知取出時間，計算各物品在該時間的溫度。第二種模式為已知目標溫度，反向計算物品需在冰箱中放置多久以及實際取出時間。

本程式採用牛頓冷卻定律作為熱傳模型。此定律描述物體溫度與環境溫度差會隨時間呈指數衰減。物品溫度可表示為冰箱溫度加上初始溫度與冰箱溫度差乘上指數衰減項。

冷卻速率與熱傳係數、接觸面積、物品質量與比熱有關。模型假設冰箱溫度為定值，物品內部溫度均勻，且僅考慮物品與冰箱環境之間的對流熱交換。

二 使用方式

執行程式後，首先輸入冰箱的溫度，單位為 Kelvin。接著輸入放入冰箱的物品數量。對每一項物品需輸入名稱、質量、比熱、熱傳係數、接觸面積、初始溫度，以及放入冰箱的時間。時間格式需為 YYYY-MM-DD HH:MM。

完成物品資料輸入後，程式會要求使用者選擇功能模式。若選擇模式一，需再輸入取出時間，程式將計算各物品在該時間的溫度，並繪製冷卻曲線圖。若選擇模式二，需輸入目標溫度，程式將計算每個物品達到該溫度所需的時間，以及實際取出時間。

程式輸出結果會以表格形式顯示在終端機中。若選擇模式一，還會顯示所有物品自放入冰箱起至取出時間的溫度變化曲線。

三 程式架構說明

程式首先定義 fridge 函式，用來根據牛頓冷卻定律計算物品在任意時間的溫度。time_check 函式負責檢查與解析使用者輸入的時間格式，以避免格式錯誤。

time_to_reach_temp 函式則用於反向計算，根據指定目標溫度求出物品所需冷卻時間。主程式負責收集使用者輸入資料，判斷使用者所選模式，並呼叫對應的計算函式。

最後，程式使用 pandas 將結果整理成表格，並在模式一中使用 matplotlib 繪製多物品的冷卻曲線圖。

四 開發過程

一開始發想是藉由化工概論提到的冰箱效率與熱傳內容而決定主題，製作一個較簡易的冰箱模擬器，只單純考量溫度變化，沒有考慮更複雜的物理現象如相變的潛熱釋放等。

最初版的程式僅計算取出末溫，但這樣我認為過於單調，於是增加繪圖的部分，將溫度變化過程的指數圖形繪出。
接著我又將程式改為可以同時投入多個物體，這樣可以比較物體間的變化過程。
第三次改動是因為想把不同圖形集合為一張圖，再利用python的datetime系統，把單純的花費時間改為真實時間，這樣觀察比較方便。
最後才把可以指定溫度的程式(第二部分)加上，讓這個程式有多一種功能可使用。

整個開發過程中有請ChatGPT教學，包含如何檢查輸入錯誤和公式詢問等，主程式未參考任何其他程式，有雷同僅為巧合。

六 程式修改與增強內容

本版本新增了由目標溫度反推取出時間的功能，使程式不僅能預測溫度，也能協助規劃取出時間。未來可進一步加入冰箱溫度隨時間變化的影響，或考慮多個物品之間的熱交換效果。

此外，程式可改為圖形化介面，或支援檔案輸入輸出，以便處理大量物品資料，並提升實際應用性。
